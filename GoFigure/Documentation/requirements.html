Go Figure Game Requirements

Overall Game

  1. Name: Main Menu
    Type: Functional
    Summary: The menu to select the level editor or the game
  2. Name: Game Layout
    Type: Functional
    Summary: This will be where levels are played
  3. Name: Game Rules
    Type: Functional
    Summary: Rules that define the game
  4. Name: Mobility
    Type: Functional
    Summary: User must be able to move game character through world through the WASD keys
  5. Name: Visbility
    Type: Functional
    Summary: User must be able to see enough of the game world to move and make decisions
  6. Name: Clock
    Type: Functional
    Summary: To have time constraints on completing a level, the game will need a method of keeping time
  7. Name: Mute Toggle
    Type: Functional
    Summary: User must be able to turn off sounds/music
  8. Name: Playable Character
    Type: Functional
    Summary: User must have a way to interact with the level(s)
  9. Name: Number of Playable Characters
    Type: Constraint
    Summary: The user will have at most 3 characters to select
  10. Name: Terrain
    Type: Functional
    Summary: This will be used as the floor or for platforms
  11. Name: Character Jumping
    Type: Functional
    Summary: The playable character must be able to maneuver up terrain
  12. Name: Character Falling
    Type: Functional
    Summary: The playable character must be able to maneuver down terrain
  13. Name: Level Border
    Type: Functional
    Summary: Playable character can't leave the dimensions of the game
  14. Name: End Objective
    Type: Functional
    Summary: A way to signal the playable character has reached the end of the level
  15. Name: Pressure Plate
    Type: Functional
    Summary: An object that can manipulate the environment and the character
  16. Name: Spikes
    Type: Functional
    Summary: An object which will kill the playable character
  17. Name: Character Death
    Type: Functional
    Summary: The playable character will be killed after one hit
  18. Name: Sound Effects
    Type: Functional
    Summary: When the playable character interacts with the environment or object, a sound will be played
  19. Name: Tutorial
    Type: Functional
    Summary: A way to tell the user about the controls and different objects in the game
  20. Name: Pre-built Levels
    Type: Functional
    Summary: At least 3 levels will already be made with the game
  21. Name: Return Button
    Type: Functional
    Summary: This button will be in game level, which will allow the user to back to the previous menu/level editor
  22. Name: Program Storage
    Type: Functional
    Summary: Game and level editor will access the same folder
  23. *New Name: Level Ending
    Type: Functional
    Summary: After level has ended let player choose to progress, quit, or restart

Level Editor

  1. Name: Level Editor - Rules
    Type: Constraint
    Summary: The level editor cannot break the rules of the game mode
  2. Name: Level Editor
    Type: Functional
    Summary: Drag and drop interface with a grid layout for the level for user to make their own levels
  3. Name: Level Editor - Testing
    Type: Functional
    Summary: The user will be able to go from the level editor to the game to test their level
  4. Name: Level Editor - Edit
    Type: Functional
    Summary: User should be able to reopen and edit previously user-made levels
  5. Name: Level Editor - Deleting
    Type: Functional
    Summary: The level editor needs a way to remove objects
  6. Name: Saved Levels
    Type: Functional
    Summary: User created levels will need to be saved
  7. Name: Level Menu
    Type: Functional
    Summary: User must be able access and play their level(s) from the game GUI
  8. Name: Level Deletion
    Type: Functional
    Summary: The user can delete levels from the level menu
  9. Name: Deleting Custom Levels
    Type: Constraint
    Summary: The user is only able to delete user created levels

Misc.

  1. Name: Physics
    Type: Risk
    Summary: Implementing the collosion detection and gravity of objects
  2. Name: Difficulty Curve
    Type: Risk
    Summary: Developing a decently fun game with user created levels that work on time
  3. Name: File Storage
    Type: Risk
    Summary: Storing the information of a level in a file
  4. Name: Music
    Type: Risk
    Summary: The management of both sound effects and music being played
  5. Name: Window Resolution
    Type: Constraint
    Summary: The windows form will be 720 x 1280px, must cooperate with visibility requirement
  6. Name: Hardware Specs
    Type: Constraint
    Summary: Physics and game calculations must be able to be processed by most computers
  7. Name: Artwork
    Type: Constraint
    Summary: Use artwork that is in the public domain
  8. Name: Free Music
    Type: Constraint
    Summary: Use Music that is in the public domain
  9. *New Name: Frame Rate
    Type: Constraint
    Summary: FPS must be 60 or higher
  10. *New Name: Response Time
    Type: Constraint
    Summary: Input should be processed in 50ms or less
