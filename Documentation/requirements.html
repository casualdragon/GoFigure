Requirements

Functional

  1. Name: Mobility
    Summary: User must be able to move game character through world
  2. Name: Visbility
    Summary: User must be able to see enough of the game world to move and make decisions
  3. Name: Clock
    Summary: To have time constraints on completing a level, the game will need a method of keeping time
  4. Name: Level editor
    Summary: Drag and drop interface for user to make their own levels
  5. Name: Saved levels
    Summary: User created levels will need to be saved, probably in a folder
  6. Name: Game and lvl editor
    Summary: Game and level editor will have to be able to access same folder
  7. Name: Level access
    Summary: User must be able access and play their level(s) from the game GUI
  8. Name: Mute toggle
    Summary: User must be able to turn off sounds/music
  9. Name: Main Menu
    Summary: The program needs a menu to select to select a level or the level editor

Constraint

  1. Name: window_resolution
    Summary: The windows form will be 720 x 1280, must cooperate with visibility requirement
  2. Name: Hardware Specs
    Summary: Physics and game calculations must be able to be processed by low end processor and 1GB RAM
  3. Name: Hardwares
    Summary: Must have a keyboard, mouse, and 720 x 1280 or higher resolution screen
  4. Name: Artwork
    Summary: Using artwork that is in the public domain
  5. Name: window_resolution
    Type: constraint
    Summary: The windows form will be 720 x 1280

Risk

  1. Name: Physics
    Summary: Game physics not breaking
  2. Name: Difficult Curve
    Summary: Developing a decently fun game with user created levels that work on time
  3. Name: Skills
    Summary: Does our team have the programming skills necessary to complete this task the way we imagine
